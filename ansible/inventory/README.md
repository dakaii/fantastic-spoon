# Ansible Inventory — Provider Contract

Every provisioner (aws-ec2, libvirt, on-prem) must produce inventory in this format.
Ansible playbooks, bootstrap scripts, and GitOps do **not** care how nodes were created.

## Required structure

```yaml
all:
  vars:
    ansible_user: <ssh-user>           # e.g. ubuntu, pi, admin
    cluster_name: primary|standby
    cluster_profile: primary|standby   # selects addon role in site.yml
    provisioner: aws-ec2|libvirt|on-prem
    k3s_version: "v1.29.5+k3s1"
    ansible_ssh_common_args: "-o StrictHostKeyChecking=no"

    # Recommended for failover / ingress setup
    ingress_host: <nlb-dns-or-ip>
    k3s_api_host: <first-control-plane-ip>

  children:
    k3s_server:
      hosts:
        <unique-hostname>:
          ansible_host: <reachable-ip-or-dns>
          node_role: server
    k3s_agent:
      hosts:
        <unique-hostname>:
          ansible_host: <reachable-ip-or-dns>
          node_role: agent
```

## Host groups

| Group | Purpose |
|-------|---------|
| `k3s_server` | Control plane nodes (1 for dev, 3 for HA) |
| `k3s_agent` | Worker nodes |

## Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `cluster_name` | Yes | Logical name: `primary` or `standby` |
| `cluster_profile` | Yes | `primary` = full addons; `standby` = lightweight |
| `provisioner` | Yes | Which Layer 1 created these nodes |
| `ingress_host` | No | Traefik/NLB endpoint for DNS failover |
| `k3s_api_host` | No | Used by Lambda witness health checks |
| `velero_bucket` | No | Set during bootstrap for backup config |

## Examples

| File | Provisioner |
|------|-------------|
| `on-prem.primary.example.yml` | On-prem primary cluster |
| `on-prem.standby.example.yml` | On-prem standby at second site |
| `primary-hosts.yml` | Generated by `primary-cluster/` Terraform |
| `standby-hosts.yml` | Generated by `cloud-services/` Terraform |

## Validation

```bash
./scripts/validate-inventory.sh ansible/inventory/on-prem.primary.yml
```
